Die Liste soll nicht einen Verein pro Spieltag haben, sondern immer nur wenn ein neuer Verein die Führung übernimmt (d.h. jeder Eintrag ist von dem vorherigen verschieden). --- # Verallgemeinerungen Wie kann man die Grundoperationen (Iterieren, Filtern, Sortieren, Abbilden, Reduzieren) verallgemeinern? $\rightarrow$ Trennung von Iterations- und Verarbeitungscode! --- # Iterieren (zum Verbrauch) ```java interface Consumer { void accept(T t); } ``` ```java static void fuerJedes(Collection coll, Consumer cons) { for (T t : coll) cons.accept(t); } ``` Beispiel: Jedes Element auf `System.out` ausgeben. ```java List li = Arrays.asList(3, 1, 3, 3, 7); fuerJedes(li, new Consumer() { public void accept(Integer i) { System.out.println(i); } }); ``` --- # Filtern ```java interface Predicate { boolean test(T t); } ``` ```java static List filtern(Collection liste, Predicate pred) { // neue Liste erstellen List gefiltert = new LinkedList<>(); // iterieren... for (T v : liste) { // Bedingung prüfen... if (pred.test(v)) gefiltert.add(v); } return gefiltert; } ``` --- # Abbilden ```java interface Function { R apply(T t); } ``` ```java static List abbilden(List liste, Function func) { List abgebildet = new LinkedList<>(); for (T v : liste) abgebildet.add(func.apply(v)); return abgebildet; } ``` --- # Reduzieren (Version 1) ```java interface BinaryOperator { T apply(T a, T b); } ``` ```java static T reduzieren(Collection liste, T identity, BinaryOperator op) { T a = identity; for (T t : liste) a = op.apply(a, t); return a; } ``` --- # Reduzieren (Version 2) ```java interface BiFunction { C apply(A a, B b); } ``` ```java static U reduzieren(Collection liste, U identity, BiFunction op) { U a = identity; for (T t : liste) a = op.apply(a, t); return a; } ``` --- # Lambdaausdrücke ```java Predicate istGerade = new Predicate () { @Override public boolean test(Integer i) { return i % 2 == 0; } }; ``` **Bedingung:** - Interface mit genau einer Methode - Interface annotiert mit `@FunctionalInterface` ```java Predicate istGerade = (i) -> { return i % 2 == 0; }; // wenn 1 Argument, 1 Anweisung: noch kompakter! Predicate istGerade = i -> i % 2 == 0; ``` --- # Lambdaausdrücke - drastische **Reduktion** von Code! - drastische **Steigerung** der Lesbarkeit! ## Was macht folgender Code? ```java List li = Arrays.asList(3, 2, 4, 1, 9, 7, 6); for (Integer i) { if (i < 8) { double c = Math.sqrt(i); if (c > 2) System.out.println(c); } } ``` --- # Lambdaausdrücke ## Mit Grundoperationen ```java List li = Arrays.asList(3, 2, 4, 1, 9, 7, 6); li = filtern(li, i -> i < 5); List ld = abbilden(li, i -> Math.sqrt(i)); fuerJedes(li, i -> System.out.println(i)); ``` ## Oder als Stream ```java Arrays.asList(3, 2, 4, 1, 9, 7, 6).stream() .filter(i -> i < 5) .map(Math::sqrt) .filter(d -> d > 2) .forEach(System.out::println); ``` Ist das nicht wirklich kurz und prägnant?!
